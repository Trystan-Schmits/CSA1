Primative Types:
This unit does not come with a specifc FRQ question, but the content shows up everywhere.
Primative types are the most basic type of variable that exist. They are stored as raw values directly onto the memory stack for quick access.
When the executing code leaves the current scoping (for example a method returns), the stack of memory corresponding to the scope is flushed by the garbage collector.
Primiative types include:

  • boolean
  • byte
  • short
  • int
  • long
  • float
  • double
  • char

    • These types take up a limited amount of memory depending on the type.
    In java Every Primative type has a Wrapper class, which is an Object that provides methods for the Primative Type.

    Java Objects (Reference Types) are stored differently than primatives Type. They use Heap Memory.
    Then after placing the Object in memory, it creates a reference and places it in the stack Memory.
    Once there are no more references to the Object, or the references are out of scope, The garbage Collector removes the Object from the memory heap.
